# KiroX 反检测增强方案

## 背景

AWS TES 风控拦截注册请求。当前 KiroX 缺失代理轮换、IP 预检、APM 追踪头和 TLS 指纹多样化。参考 openai-cpa-17.0.3 制定以下改进方案。

## 方案总览

| # | 模块 | 改动文件 | 难度 | 说明 |
|---|------|---------|------|------|
| 1 | Clash 代理轮换 | `internal/proxy/clash.go` | 🔴 | 注册前自动切节点，过滤港台 |
| 2 | IP 预检 | `internal/proxy/precheck.go` | 🟢 | 注册前查 cloudflare trace |
| 3 | Datadog 追踪头 | `internal/http/helper.go` | 🟢 | 注入 6 个 APM 追踪头 |
| 4 | TLS 多样化 | `internal/http/helper.go` | 🟢 | ChromeVer 映射不同 profile |
| 5 | 指纹刷新 | `internal/core/registrar.go` | 🟢 | 每账号生成新身份 |
| 6 | 前端配置 UI | `frontend/` + `config.go` | 🟡 | Clash 配置 + 反检测开关 |
| 7 | 预检集成 | `internal/task/coordinator.go` | 🟢 | startTask 时执行 IP 检查 |
| 8 | 验证 | 全量编译 | 🟢 | 向后兼容确认 |

## 详细设计

### 1. Clash 代理轮换模块

文件: internal/proxy/clash.go

ClashRotator 结构体:
- APIURL / Secret / GroupName / Blacklist / FastestMode / CurrentProxyURL
- SwitchNode(fallbackProxy) 方法：获取节点延迟 → 过滤黑名单 → 最快或随机选 → PUT 切换 → 返回代理 URL
- 无 Clash 时 (APIURL 为空) 直接返回 fallbackProxy

默认黑名单: 自动、故障、港、HK、台、TW、CN 等 18 个关键词
默认策略组: 节点选择

### 2. IP 预检

文件: internal/proxy/precheck.go

PreCheckProxy(proxyURL string) error
- 代理为空时跳过
- 访问 cloudflare.com/cdn-cgi/trace
- 解析 loc/ip/colo，日志输出
- CN/HK 返回 ErrRegionBlocked（警告不阻断）

### 3. Datadog 追踪头

6 个标准 APM 追踪头:
- traceparent: 00-{32hex}-{16hex}-01
- tracestate: dd=s:1;o:rum
- x-datadog-origin: rum
- x-datadog-parent-id / x-datadog-sampling-priority / x-datadog-trace-id

注入点: http_helpers.go 的 _get/_post/_put/_delete，kiro_auth.go
每个新会话生成全新 trace ID

### 4. TLS Profile 多样化

ChromeVer → TLS profile 映射:
120 → Chrome_120
121-124 → Chrome_124
125-129 → Chrome_128
130-134 → Chrome_131
135-139 → Chrome_136
140-144 → Chrome_144

NewTLSClient 接收 chromeVer 参数做映射

### 5. 每账号刷新指纹

每次 NewRegistrar 时调用 RandomIdentity() 而非缓存 IdentityForProxy
配置项 refresh_fingerprint_per_account (默认 true)

### 6. 前端 UI

Config 新增 ClashConfig + AntiDetectConfig 结构体
前端添加 Clash 配置面板和反检测开关

### 7. 预检集成

startTask 中首次调用 PreCheckProxy，日志输出不阻塞

### 8. 验证

go build ./... 通过，向后兼容
